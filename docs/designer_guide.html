1 Creating & Editing Messes

The apartment gets messy each morning based on narrative conditions tied to date outcomes, flower trims, and day number. Each mess is a MessBlueprint ScriptableObject.

flowchart LR A["MessBlueprint SO"] --> B{"AuthoredMessSpawner\n(day start)"} B -->|conditions pass| C["Spawn Stain\nor Object"] B -->|conditions fail| D["Skip"] E["DateOutcomeCapture"] --> B F["GameClock.CurrentDay"] --> B

Using the Mess Editor

  1. Open Window > Iris > Mess Editor
  2. Browse all blueprints in the left panel — filter by Category (DateAftermath / OffScreen / General) or Type (Stain / Object)
  3. Click any blueprint to edit it inline in the right panel
  4. Toggle Scene Gizmos to see placement positions in Scene View (discs = stains, cubes = objects)
  5. Use "Capture Position from Scene View" button to set spawn position from your current Scene camera

Creating a New Mess

  1. Click "New Mess Blueprint" in the Mess Editor, or right-click in Project > Create > Iris > Mess Blueprint
  2. Save to Assets/ScriptableObjects/Messes/
  3. Set the Classification: Category + Type
  4. Set Conditions (see table below)
  5. Set Placement: position, rotation, allowed areas, weight
  6. For Stains: assign a SpillDefinition
  7. For Objects: assign a prefab, or leave null for a procedural colored box

Condition Reference

FieldEffect
requireDateSuccessOnly appears after a successful date
requireDateFailureOnly appears after a failed date
minAffectionMinimum affection score from last date
requireReactionTagDate must have reacted to this specific tag (e.g. "wine")
requireBadFlowerTrimLast flower score was < 40
requireGoodFlowerTrimLast flower score was ≥ 80
minDayEarliest day this mess can appear
allowedAreasWhich apartment areas it can spawn in
weightHigher = more likely to be picked (weighted random)
Gizmo colors in Scene View: Red = DateAftermath, Blue = OffScreen, Green = General

2 Placeable Objects — Home & Disheveled Rotations

Every PlaceableObject has a home rotation (normal/tidy) and an optional disheveled rotation (messy/knocked over). The tidiness system uses these to score apartment cleanliness.

stateDiagram-v2 [*] --> Normal : Scene Start Normal --> Disheveled : Dishevel() Disheveled --> Normal : Straighten() / Pickup Normal --> Held : Player grabs Held --> Normal : Player places Disheveled --> Held : Player grabs

Setting Up Rotations

  1. Select the object in the scene
  2. Orient it in its normal/tidy position
  3. In the Inspector, click the "Capture Home Rotation" button at the bottom of the PlaceableObject component
  4. Now rotate the object to its messy/knocked-over orientation
  5. Click "Capture Disheveled Rotation"
  6. Rotate the object back to normal (or check Start Dishelved to have it start messy)

Key Fields

FieldPurpose
_canBeDishelvedEnable the dishevel system for this object
_dishevelAngleAngle threshold — tilted beyond this = messy
_startDishelvedBegin the scene in the disheveled pose
_useSpawnAsHomeAuto-capture position at scene start as home
CategoryItem type: General, Trash, Dish, Shoe, Coat, Hat, etc.
The capture buttons only save rotation, not position. Use Capture Home Position + Rotation (context menu) if you also need to save position.

Context Menu Options

Right-click the PlaceableObject component header for additional options:

  • Capture Home Position + Rotation — saves both pos + rot
  • Capture Home Rotation — rotation only
  • Capture Disheveled Rotation — rotation only
  • Clear Disheveled Rotation — removes disheveled pose

3 Watering Plants

Each pot in the apartment has a WaterablePlant component driven by a PlantDefinition ScriptableObject.

Swapping Plant Type

  1. Select the pot GameObject in the scene
  2. On the WaterablePlant component, find the Definition field
  3. Drag a different PlantDefinition SO onto it (or create a new one)

Creating a New Plant Type

  1. Right-click in Project > Create > Iris > Plant Definition
  2. Save to Assets/ScriptableObjects/
  3. Set identity: plantName, description
  4. Set colors: potColor, dryColor, wetColor, plantColor
  5. Tune watering: idealWaterLevel, waterTolerance, pourRate
  6. Tune scoring: baseScore
FieldWhat It Controls
idealWaterLevelTarget fill line (0–1). Player aims for this.
waterToleranceHow close to ideal counts as perfect (0.01 = strict, 0.2 = forgiving)
pourRateWater units per second while pouring
targetOscSpeedHow fast the target line moves (oscillations/sec)
targetOscAmplitudeHow far the target swings above/below ideal

4 Creating a Date Character

Each date NPC is a DatePersonalDefinition ScriptableObject with preferences, timing, flower gift, and newspaper ad content.

flowchart TB A["DatePersonalDefinition SO"] --> B["NewspaperManager\n(ad display)"] A --> C["DateSessionManager\n(3-phase date)"] A --> D["ReactionEvaluator\n(item judging)"] A --> E["FlowerTrimmingBridge\n(flower scene)"] subgraph Preferences F["likedTags / dislikedTags"] G["likedDrinks / dislikedDrinks"] H["preferredMoodMin/Max"] I["likedOutfitTags"] end A --> Preferences

Creating a New Date

  1. Right-click in Project > Create > Iris > Date Personal
  2. Save to Assets/ScriptableObjects/Dating/
  3. Fill in Identity: characterName, adText
  4. Set Timing: arrivalTimeSec
  5. Assign Visuals: portrait, characterModelPrefab
  6. Configure Preferences (the sub-object):
    • likedTags / dislikedTags — matched against ReactableTag on apartment items
    • likedDrinks / dislikedDrinks — DrinkRecipeDefinition refs
    • preferredMoodMin / preferredMoodMax — ideal mood range (0=sunny, 1=stormy)
    • likedOutfitTags / dislikedOutfitTags
    • clutterTolerance — 0 = hates mess, 1 = doesn't care
  7. Add Keywords: hoverable words in the newspaper ad with tooltips and prep tags
  8. Set Flower Gift: flowerSceneName (scene in Build Settings), flowerPrefab
  9. Add to a NewspaperPoolDefinition SO for it to appear in the newspaper rotation
guaranteeFlowerSuccess = true makes the flower trim always score high — use for tutorial dates.

5 Records & Music

Records are click-to-select — no physical grabbing. Click a record on the shelf to send it to the turntable. Click the turntable to eject.

Creating a New Record

  1. Right-click in Project > Create > Iris > Record Definition
  2. Save to Assets/ScriptableObjects/RecordPlayer/
  3. Set title, artist, labelColor, coverColor
  4. Assign musicClip (AudioClip) and volume
  5. Set moodValue (0 = sunny, 1 = stormy — feeds MoodMachine)
  6. In the scene, create a record mesh with PlaceableObject + RecordItem components
  7. Drag the RecordDefinition SO into RecordItem's Definition field
FieldEffect
moodValue0 = pushes mood sunny, 1 = pushes mood stormy. Affects lighting, fog, rain.
labelColorColor of the spinning disc label on the turntable
volumePlayback volume (0–1)
Set Starting Record on the RecordSlot component to have a record playing when the scene loads.

6 Books

Creating a New Book

  1. Right-click in Project > Create > Iris > Book Definition
  2. Save to Assets/ScriptableObjects/Bookcase/
  3. Set title, author, spineColor
  4. Set heightScale (0.5–1.0) and thicknessScale (0.02–0.06)
  5. Add reactionTags for date NPC reactions (e.g. "poetry", "manga")
  6. Optional: set hasHiddenItem = true to drop a hidden item on first pickup
  7. In scene: add PlaceableObject + BookItem, assign the BookDefinition

7 Camera Angles & Presets

Camera presets define different visual directions per area. Each preset is a CameraPresetDefinition SO with per-area camera position, lens settings, post-processing, and light overrides.

flowchart TB A["CameraPresetDefinition SO"] --> B["AreaCameraConfig[]"] B --> C["Area 0: Kitchen"] B --> D["Area 1: Living Room"] B --> E["Area 2: Entrance"] C --> F["position, rotation\nLensSettings (FOV)\nVolumeProfile\nlight overrides"] D --> F E --> F

Creating a New Camera Preset

  1. Right-click in Project > Create > Iris > Camera Preset
  2. Save to Assets/ScriptableObjects/Apartment/
  3. Set a label (e.g. "V1", "Cinematic", "Top-Down")
  4. For each area (Kitchen, Living Room, Entrance), set:
    • Position & Rotation — camera transform
    • Lens — FOV, near/far clip, dutch angle
    • Volume Profile — URP post-processing (bloom, color grading, DoF, vignette)
    • Light Intensity Multiplier — scales directional light (on top of MoodMachine)
    • Light Color Tint — tints directional light
  5. Select the preset SO in Scene View to see frustum wireframe gizmos
  6. Use "Capture Scene View" buttons per area in the custom inspector to snap from your current Scene camera

Runtime Controls

KeyAction
1 / 2 / 3Apply preset 1, 2, or 3
` (backtick)Clear preset (return to default)
Two-layer lighting: Layer 1 = MoodMachine (player actions → global mood). Layer 2 = Camera preset (VolumeProfile + light overrides). They stack.

8 Creating New Flowers

The Quick Flower Builder wizard auto-generates a play-ready flower hierarchy from mesh inputs, complete with physics, scoring rules, and ScriptableObjects.

flowchart LR A["Stem Mesh\nLeaf Mesh\nPetal Mesh"] -->|Quick Flower Builder| B["Flower Hierarchy"] B --> C["FlowerGameBrain\n+ SessionController"] B --> D["IdealFlowerDefinition SO"] B --> E["FlowerTypeDefinition SO"] B --> F["Physics:\nRigidbody\nXYTetherJoint\nSquishMove"]

Using the Quick Flower Builder

  1. Open Window > Iris > Quick Flower Builder
  2. Drag your stem, leaf, and petal meshes into the source fields
  3. Optionally drag a crown mesh
  4. Set leafCount and petalCount
  5. Set flowerName and difficulty
  6. Adjust arrangement: leafRadius, petalRadius, height ranges
  7. Set ideal trimming: idealStemLength, idealCutAngle
  8. Click "Build Flower"

What Gets Created

{flowerName}_Flower (root)
  ├── Stem (FlowerStemRuntime, Rigidbody)
  │   ├── StemAnchor
  │   ├── StemTip
  │   └── CutNormalRef
  ├── Crown (FlowerPartRuntime, kind=Crown)
  ├── Leaf_0 ... Leaf_N (FlowerPartRuntime, kind=Leaf)
  └── Petal_0 ... Petal_N (FlowerPartRuntime, kind=Petal)

ScriptableObjects:
  Assets/ScriptableObjects/Flowers/Ideal_{name}.asset
  Assets/ScriptableObjects/Flowers/Type_{name}.asset

Manual Flower Setup

If you prefer to build by hand instead of using the wizard:

  1. Create root GO with FlowerGameBrain + FlowerSessionController
  2. Add stem child with FlowerStemRuntime + Rigidbody
  3. Add leaf/petal children with FlowerPartRuntime + Rigidbody + XYTetherJoint
  4. Set unique PartId on each FlowerPartRuntime (must match the IdealFlowerDefinition rules)
  5. Create IdealFlowerDefinition and FlowerTypeDefinition SOs
  6. Wire: Brain → ideal + stem + parts, SessionController → brain + FlowerType
  7. Click "Validate Existing" in Quick Flower Builder to check wiring

9 Flower Difficulty Levels

Different "levels" of the same flower are created by pairing the same flower prefab with different IdealFlowerDefinition SOs that have tighter or looser tolerances.

flowchart TB A["Same Flower Prefab\n(Daisy)"] --> B["Easy: Ideal_Daisy_Easy"] A --> C["Normal: Ideal_Daisy_Normal"] A --> D["Hard: Ideal_Daisy_Hard"] B --> E["Loose tolerances\nstemHardFailDelta: 0.5\ncutAnglePerfectDelta: 15"] C --> F["Medium tolerances\nstemHardFailDelta: 0.3\ncutAnglePerfectDelta: 8"] D --> G["Tight tolerances\nstemHardFailDelta: 0.15\ncutAnglePerfectDelta: 3"]

Creating a Harder/Easier Version

  1. Duplicate an existing IdealFlowerDefinition SO (e.g. Ideal_Daisy.asset → Ideal_Daisy_Hard.asset)
  2. Adjust the tolerance deltas (see table below)
  3. Create a new FlowerTypeDefinition SO pointing to the new ideal
  4. Set difficulty enum and adjust scoreMultiplier, minDays, maxDays
  5. In the flower scene, swap FlowerSessionController.FlowerType to the new type

Difficulty Tuning Reference

ParameterEasyNormalHard
stemPerfectDelta0.20.080.02
stemHardFailDelta1.00.40.15
cutAnglePerfectDelta15°8°3°
cutAngleHardFailDelta60°35°15°
allowGameOverfalsetruetrue

Score → Days Mapping

By default, score maps linearly to days: days = Lerp(minDays, maxDays, normalizedScore)

For non-linear mapping (e.g. punish mediocre scores): set useCustomCurve = true and edit the scoreToDaysCurve AnimationCurve. X axis = normalized score (0–1), Y axis = normalized days (0–1).

Per-Part Rules
FieldPurpose
partIdMust match FlowerPartRuntime.PartId exactly
kindLeaf / Petal / Crown
idealConditionNormal / Perfect / Withered
canCauseGameOverRemoving this part = instant fail (Crown typically true)
isSpecialMarks "boss" leaves that trigger fail if removed
allowedWitheredWithered state is acceptable (scores 0.5)
allowedMissingMissing is acceptable (scores 0.5)
scoreWeightRelative importance in final score

10 Mood Machine Profiles

The MoodMachine blends all active sources (perfume, record, time of day, plants) into a single mood value (0=sunny, 1=stormy) and applies a MoodMachineProfile.

flowchart LR P["Perfume"] -->|SetSource| M["MoodMachine"] R["Record"] -->|SetSource| M T["Time of Day"] -->|SetSource| M L["Living Plants"] -->|SetSource| M M -->|mood 0-1| Prof["MoodMachineProfile"] Prof --> Light["Directional Light\ncolor + intensity"] Prof --> Amb["Ambient Color"] Prof --> Fog["Fog Color + Density"] Prof --> Audio["Ambience + Weather Volume"] Prof --> Rain["Rain Particles"]

Creating / Editing a Profile

  1. Right-click in Project > Create > Iris > Mood Machine Profile
  2. Edit the Gradients (light color, ambient, fog) and AnimationCurves (intensity, density, volumes)
  3. Assign to the MoodMachine component in the apartment scene

11 Wall Dissolve (See-Through Walls)

Walls can dissolve per camera area so the player can see into rooms. Uses the Iris/PSXLitDissolvable shader.

Making a Wall See-Through for a Specific Area

  1. Assign the PSXLit_Dissovable material to the wall renderer
    Assets/Materials/PSXLit_Dissovable.mat
  2. Ensure the wall has a collider on the correct layer
  3. Add the AreaWallFade component
  4. In Area Dissolves array, add an entry:
    • Area Index: 0 = Kitchen, 1 = Living Room, 2 = Entrance
    • Dissolve Amount: 0.85 = mostly invisible (tweak to taste)
  5. Ensure WallOcclusionFader exists in the scene (singleton)
For always-dissolved walls (isometric cutaway style): use AlwaysFadedWall instead — set BaseDissolve (e.g. 0.6).
Wall-mounted items won't dissolve their own wall — the system automatically exempts the wall behind the target.

12 Drink Recipes

Creating a New Drink

  1. Right-click in Project > Create > Iris > Drink Recipe
  2. Save to Assets/ScriptableObjects/DrinkMaking/
  3. Set drinkName, requiredGlass, liquidColor
  4. Add ingredients and portionNormalized arrays (should sum to ~1.0)
  5. Set idealFillLevel (0–1) and fillTolerance
  6. Optional: requiresStir + stir speed range
  7. Link to date's likedDrinks / dislikedDrinks in their DatePreferences
